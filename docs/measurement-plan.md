# P.S.parkle 計測設計

## サイト内で自動計測するイベント

| イベント名 | 内容 | 主な発火場所 |
|---|---|---|
| `line_click` | 公式LINEへの遷移 | ヘッダー、FV、各ページ最下部、スマホ固定CTA、フッター |
| `calendar_click` | 予約カレンダーへの遷移 | FV、各ページ最下部 |

各イベントには、ボタン位置を示す `cta_location` と遷移先を示す `destination` が付与されます。

## Google Analytics 4の接続

1. Google AnalyticsでGA4プロパティとWebデータストリームを作成する
2. 測定ID（`G-` から始まるID）を取得する
3. 公開サービスの環境変数に `NEXT_PUBLIC_GA_MEASUREMENT_ID` を登録する
4. 再ビルド・再公開する
5. GA4のリアルタイム画面で `line_click` と `calendar_click` を確認する
6. 両イベントをキーイベントとして設定する

## LINE追加後から入会までの管理

サイトだけで判定できるのは「LINEボタンを押した」「カレンダーを押した」までです。実際のLINE追加、体験予約、体験参加、入会は公式LINE側で次のタグを付けて月次集計します。

| LINEタグ | 定義 |
|---|---|
| `HP流入` | HP経由と確認できた友だち追加・問い合わせ |
| `体験案内済` | 無料体験の候補日時を案内した |
| `体験予約` | 日時・会場が確定した |
| `体験参加` | 体験レッスンに参加した |
| `入会` | 月額コースへ入会した |
| `見送り` | 今回は入会しなかった |

## 月次KPI

| 指標 | 目標 |
|---|---:|
| 問い合わせ | 7件以上 |
| 体験予約 | 5〜10件 |
| LINE追加 | 15〜30件 |
| 体験ページの予約クリック率 | 5%以上 |
| 体験から入会への転換率 | 50%以上 |

問い合わせ数は、GA4のクリック数だけではなく、公式LINEで実際に返信があった人数を正として管理します。
